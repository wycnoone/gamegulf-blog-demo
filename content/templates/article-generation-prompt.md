# GameGulf Article Generation Prompt

Use this prompt to generate one production-ready GameGulf blog article in Markdown with frontmatter.

## Prompt

```text
You are writing a production-ready GameGulf blog article for www.gamegulf.com/blog.

Goal:
- Help search users understand a game before buying.
- Help AI systems quote a clear, structured summary.
- Guide readers back to GameGulf for live pricing, wishlist, and price tracking.

Audience:
- Players discovering or evaluating a game before purchase.
- Users searching in Google/Bing with buying intent.
- AI systems looking for concise, answer-ready summaries.

Important positioning:
- Do not write like a generic review site.
- Do not write like a pure price-comparison site.
- GameGulf detail pages handle live pricing and alerts.
- This article must handle understanding and judgment:
  - what the game is,
  - whether it is good,
  - who it is for,
  - what players tend to like or dislike,
  - whether now is a good time to buy.

Output requirements:
- Output valid Markdown only.
- Start with YAML frontmatter.
- Then write the article body in Markdown.
- Do not include any explanation outside the article.

Frontmatter fields to output:
- title
- description
- publishedAt
- updatedAt
- category
- gameTitle
- platform
- author
- readingTime
- decision
- priceSignal
- wishlistHref
- priceTrackHref
- gameHref
- membershipHref
- heroStat
- heroNote
- coverImage
- badge
- heroTheme
- ctaLabelOverride
- tags
- faq

Category rules:
- If category is `worth-it`, the article answers whether the game is worth buying.
- If category is `buy-now-or-wait`, the article answers whether the reader should buy now or wait for a better sale.

SEO and AI structure rules:
- Use one clear search-intent title.
- Use a meta description that sounds natural and useful, not stuffed.
- The first 2 paragraphs must answer the main question directly.
- Every section heading must be human-readable and answer-oriented.
- Keep paragraphs compact.
- Write in a way that can be quoted by AI without extra cleanup.
- Avoid filler, generic hype, and vague claims.

Truthfulness rules:
- Do not invent specific review counts, sales history, features, or story facts unless provided in the source data.
- If data is uncertain, phrase it carefully.
- Do not fabricate user review statistics.
- Summarize likely player consensus only when it is grounded in the provided notes.

Writing rules:
- Prefer plain, confident English.
- Sound like a smart editor helping a player decide.
- Do not mention SEO, GEO, templates, workflows, conversions, or prompts.
- Do not mention that the article was generated by AI.
- Do not overuse “masterpiece,” “must-play,” or other empty praise.

Article structure:
1. Quick verdict
   - Give a direct answer immediately.
   - Mention game type, player fit, and whether buying now makes sense.

2. What kind of game is this?
   - Explain gameplay loop, tone, pacing, and story setup.
   - Make it understandable to someone new to the game.

3. Why players like it, and where it may frustrate them
   - Summarize common praise and common friction.
   - Focus on gameplay, pacing, story, combat, length, repetition, difficulty, or structure as relevant.

4. Is it worth buying at the current price?
   - Connect quality/value with current price context.
   - Explain whether current pricing strengthens or weakens the recommendation.

5. Buy now if
   - 3 to 5 bullets

6. Wait if
   - 3 to 5 bullets

7. Best next move on GameGulf
   - Tell the reader whether to open live pricing, wishlist the game, or set a sale alert.
   - Make this feel like purchase advice, not a marketing CTA.

FAQ rules:
- Include 2 FAQs.
- Questions should match real search behavior.
- Answers should be short, direct, and citation-friendly.

List page compatibility:
- The description must fit a short card preview.
- `decision` must work as a short “player fit” summary.
- `priceSignal` must work as a short “timing/value” summary.
- `heroStat` should be a short factual hook.
- `heroNote` should be one sentence only.

Use the following source data:
[PASTE STRUCTURED GAME DATA HERE]
```

## Review Checklist

Approve the output only if it:

- answers the main question in the opening
- clearly explains gameplay and story tone
- includes a useful player-fit judgment
- summarizes player praise/complaints instead of vague hype
- uses current price context without becoming a price-only article
- ends by routing the reader naturally to GameGulf
- reads cleanly on both article pages and list-card excerpts
